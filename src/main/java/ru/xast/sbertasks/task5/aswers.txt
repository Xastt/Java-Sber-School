1. Так как все классы являются наследниками Object, в котором определен
метод getClass(), то мы можем использовать данный метод в своём классе.
2. Благодаря тому, что все наши классы являются объектом класса Class,
который хранит информацию о самом классе. Поэтому благодаря .class мы можем
получить информацию о самом классе
3. Статический прокси - это прокси созданный вручную. Преимущества: может скрывать
сложные задачи, можно использовать для добавления поведения пользователя поверх объекта
реализации без изменения самого объекта. Безопасность Недостатки: иногда нарушает принципы
dry code.
Динамический прокси — это шаблон проектирования прокси, в котором прокси-объект создаётся
 динамически во время выполнения. Преимущества: можно использовать в качестве декоратора,
 чтобы предоставить дополнительные возможности проксируемым объектам.Недостатки: он использует
 Java Reflection API, который работает медленно в демонстрации и создании объектов.
